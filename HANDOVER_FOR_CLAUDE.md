# 引き継ぎ書: FreeTierWatch 運用司令部

**宛先**: この文書を読んでいるClaude(Opus または Sonnet)へ。あなたは前任のFable 5からこのプロジェクトの司令塔を引き継ぎます。この文書だけで全体を理解し、運用を指揮できるように書いてあります。

## オーナーについて

- Stellaさん。**英語は読めません。報告・説明はすべて日本語で。**
- プログラミング知識はほぼゼロ。専門用語は避け、クリック単位で案内すること
- 目的: 維持費0円で動き続ける収益資産の運用。確実に儲かる話ではないことは本人も了解済み

## プロジェクトの全体像

- **サイト**: https://aiu24rrf-hub.github.io/freetierwatch/ (AI APIの無料枠・料金を毎日自動検証して公開する英語メインのトラッカー)
- **リポジトリ**: https://github.com/aiu24rrf-hub/freetierwatch (公開。git cloneで読める)
- **自動化(GitHub Actions側で完結、Claude不要)**:
  - 毎朝6:17 JST「Daily data verification」: 15サービスの公式ページを取得→Gemini無料枠APIで変更検出→data/*.jsonを更新してコミット
  - 毎週火曜7:00 JST「Weekly post drafts」: 週の変更まとめからX投稿文(英日)を生成し、GitHub Issueとして起票(オーナーにメール通知が飛ぶ)
- **収益導線**: Ko-fi(投げ銭)、Gumroad(The $0 AI Stack Playbook、$12のPDF)、将来はリファラル/スポンサー
- **設計原則(必ず守る)**:
  1. 維持費は月0円を守る。ランタイムでClaude APIを呼ぶ構成に変えない
  2. 失敗しても前日データを保持するfail-soft設計を壊さない
  3. HN/Redditへの自動投稿は絶対にしない(BAN=資産喪失)。投稿は必ず本人の手で
  4. **司令塔(あなた)は思考と判断のみ。調査・コード作業・長文生成はSonnet/Haiku等の低コストサブエージェントに委任してコストを節約する**

## 定例業務

### 週次(毎週火曜、自動のスケジュールタスクが起動する)
1. `git clone https://github.com/aiu24rrf-hub/freetierwatch.git` して data/changelog.json の直近7日を確認
2. WebFetchでサイトが200で表示されるか確認
3. X投稿文2本(英語+日本語)を完成形で作成(変更ゼロの週は「今週は変更なし=安定」の切り口)
4. 日本語で報告: 状態(正常/異常)+コピペ用投稿文。異常時は下のトラブル対応表に従い修正手順を提示

### 月次(毎月、自動のスケジュールタスクが起動する)
1. data/services.json の meta.last_updated が3日以内か確認(古い=自動更新が止まっている)
2. index.html の Ko-fi/Gumroad リンクがプレースホルダのままなら、TODAY_DO_JA.md の該当手順を再案内
3. 改善提案を1つだけ(追跡サービス追加、SEO、記事ネタ等)。やるかはオーナーが判断

## トラブル対応表

| 症状 | 原因の見立て | 対処 |
|---|---|---|
| サイトが404 | Pages設定が外れた | Settings→Pages→main/(root)を再設定するよう日本語で案内 |
| last_updatedが古い | Actionsの失敗/無効化 | Actionsタブのログを本人にスクショしてもらい、エラーを読んで修正。60日無活動で止まった場合はRun workflowを1回押してもらう |
| Gemini無料枠の変更でエラー | モデル名/квота変更 | scripts/update_data.py の GEMINI_MODEL を現行の無料モデル名に変更(GitHubの鉛筆編集で1行) |
| 修正が必要なファイルがある | — | 修正版の全文を生成して渡し、GitHubの鉛筆編集 or Upload filesで差し替えてもらう(git pushはこの環境から不可の場合が多い) |

## 収益の判断基準(1〜3ヶ月後)

- 反応が出たら: Zenn/noteに日本語解説記事(Gumroad導線)、追跡サービス拡充、HN/Reddit再挑戦
- 反応ゼロでも: 維持費0円なので撤退判断は不要。週次投稿だけ続けるか、頻度を落とす
- 絶対にやらない: 有料インフラへの移行、AI量産コンテンツサイト化、自動トレーディング

## 経緯の要点

2026-07-12、Fable 5最終日に構築。市場調査の結論:「AIは作ることを無料にしたが、売る(集客)は自動化できない。維持費0円の資産は放置しても損をしない」。この思想がすべての設計判断の根拠です。
